CAT Platform
02 · what it is
What this is

A publication, not a directory.

The Platform documents programmes that are actually changing how India grows, eats, and sustains. Programmes are read, not pitched. Limitations sit beside achievements. Curated by CAT, open to the broader food-systems fraternity.

In one line

A collaborative platform working to enable agroecological transformation at scale across India — surfaced for funders, NGOs, researchers, and policy actors as a single, defensible reading surface.

03 · what it is not
What this is NOT

Not a tool. Not a marketing site.

  • Not an internal project-management tool. CAT does not run its operations from here.
  • Not a GIS platform. The map is an editorial illustration that happens to be data-accurate, never a survey-grade GIS.
  • Not a CAT marketing site. CAT is the editor, not the subject.
  • Not a national authority. It is CAT's contribution to the food-systems fraternity.
  • Not a volume-driven directory. Curation is the win condition, not scale.
The reflex-reject

It must not look like a SaaS dashboard, an observability tool, a Notion page, a Medium article, a generic Tailwind starter, an AI-chat product, or a fintech landing page. It is an editorial publication: The Atlantic meets Our World in Data meets Stripe Press.

05 · standing rules
Standing rules · non-negotiable

The bar is honesty, not affiliation.

  • Curated, not crowdsourced.
  • Programme level, never activity level.
  • “What did not work” is required design infrastructure.
  • AI scales production; editors decide what publishes.
  • Funders never register.
  • Honest provenance, public last-reviewed dates.
  • The map is not GIS.
  • Plain language. No marketing words. No em dashes.

15 · levers
CAT's strategic frame · surfaced on /about + every landscape

Three levers for change.

CAT enables agroecological transformation by aligning three levers — policy, markets, and finance — to support community-led adoption and long-term transitions.

01

Policy

Schemes that work for smallholders. Regulation that opens space for agroecology. Institutions that hold the system together over many years.

02

Markets

Procurement at fair prices. Processing and aggregation that earn the producer a margin. Traceability that survives the broker chain.

03

Finance

Patient capital. Blended financing for landscape-scale work. Working-capital instruments built around farmer-collective realities, not commercial banking norms.

7-year horizon per landscape15 / 22

16 · loop
How an entry gets from a public source to a published page

The human-AI editorial loop.

01 · weekly

Source registry

80-150 curated trusted URLs (state agri depts, NABARD/IFAD, partner reports). Crawled, hashed, diffed.

02 · weekly

Discovery agent

Claude with web search, allowlisted to .gov.in / .org / news. Surfaces new candidate programmes.

03 · on demand

Draft writer

Kimi reads sources, drafts the five narrative blocks with citation anchors. Editor sees source ↔ draft side-by-side.

04 · CAT editor

Approval

Nothing publishes without a human approval. Every change recorded in entry_revisions.

Daily freshness sweep: detects source change, flags entry needs_update, never edits public prose silently

17 · stack
Locked decisions

The stack in one page.

Framework
Next.js 15 App Router · TypeScript strict · Tailwind v3
SSR for SEO on entries, single repo for frontend + API
Database
Supabase Postgres · isolated cat schema
ap-south-1 Mumbai region. Dedicated cat_user role, scoped.
ORM
Drizzle ORM + pg driver
Type-safe schema-as-code. pg chosen for IPv6 support.
Primary LLM
Kimi K2.6 via NVIDIA's hosted API
Free tier. Powers /agent preview + draft writer.
Secondary LLM
Anthropic Claude Sonnet 4.6
Only for the discovery agent — needs web_search tool.
Auth
Auth.js magic-link via Resend
No passwords. Role flag on user (reader / contributor / editor / admin).
Search
Postgres FTS + GIN index · pgvector ready
No Elasticsearch, no Algolia. Semantic synthesis is v2.
Map
Custom SVG · real Datameet/jbrobst admin-1 GeoJSON · d3-geo
No Leaflet, no Mapbox. Illustrative, not GIS.
Hosting
Vercel + cron schedules
Single platform. Cron fires registry / discovery / freshness jobs.
Cost
Free tiers across the board for v1
Supabase free · Vercel free · Kimi free · only Anthropic paid for discovery.

21 · PRD-as-questions
For the deeper walkthrough

Every decision, reverse-engineered as a question.

A companion PRD-questions doc enumerates every locked decision (positioning, content sourcing, AI authoring depth, freshness, endorsement tiers, map fidelity, palette, motion, three levers, eleven landscapes, etc.) and phrases each one back as a question for the team to confirm or revise.

docs/PRD-questions.md · 14 sections · ~40 decisions

Sample questions inside
  • Should the v1 release be the full surface or a slimmer read-only set?
  • How deep does the AI write each entry? Where exactly does it stop?
  • Are the three endorsement tiers right, and is the language right?
  • Should we surface the seven-year horizon in public copy?
  • Is this Platform the “Bharat Agroecology Tracker” CAT advertised, or a sibling?
  • What domain do we publish at?

The point of the doc: the team should disagree with at least 3-5 items, because that disagreement is where the strongest signal lives.
